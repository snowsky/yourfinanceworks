## Core Licensing Infrastructure

### Backend Services
- Add LicenseService for license validation and feature management
  * JWT-based license key verification with RSA signatures
  * 30-day trial period with 7-day grace period
  * Personal vs business usage type selection
  * Feature availability checks with environment variable fallback
  
- Add FeatureConfigService for centralized feature configuration
  * Define 13 licensable features across AI, integration, and advanced categories
  * Support for environment variable overrides (FEATURE_*_ENABLED)
  * Three-tier priority: license check → env var → default value
  
- Add feature_gate utilities with decorators
  * @require_feature decorator for API endpoint protection
  * @require_business_license for business-only features
  * HTTP 402 (Payment Required) responses for unlicensed features
  * Automatic fallback to env vars for fresh installations

### Database Schema
- Add InstallationInfo model to models_per_tenant.py
  * Track installation ID, trial dates, license status
  * Store licensed features and activation metadata
  * Support license validation caching (1-hour TTL)
  * Personal vs business usage type tracking

### API Endpoints
- Add /api/v1/license router with comprehensive endpoints:
  * POST /select-usage-type - Choose personal or business use
  * GET /status - Get current license and trial status
  * POST /activate - Activate license keys
  * POST /validate - Validate license keys without activation
  * GET /features - Public endpoint for feature metadata
  * POST /deactivate - Deactivate current license

## Migration Notes

For existing installations:
1. Run database migrations to add InstallationInfo table
2. Execute deploy_licensing_system.sh to initialize license system
3. Existing users will be granted 30-day trial automatically
4. Personal use installations get all features free forever
5. Business use installations require license after trial
